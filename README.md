# rooms

Микросервис управления комнатами платформы GAZ.

## Описание

Управляет комнатами активностей: создание, поиск, вступление и выход. Применяет ограничения по подписке на количество создаваемых комнат и участников. Валидирует JWT и получает данные пользователя через gRPC из сервиса `auth_users`.

## Архитектура

Один REST-сервер на базе PostgreSQL. Интегрируется с `auth_users` через два gRPC-клиента:

- **Token gRPC** — валидация JWT на каждом защищённом запросе
- **User gRPC** — проверка верификации email и уровня подписки перед созданием комнаты

## Технологии

- **Go 1.25** — структурированные логи через стандартный `log/slog`
- **Gin** — HTTP-роутинг, middleware `gin.Recovery()`
- **PostgreSQL** + **pgx/v5** — пул соединений, миграции через `golang-migrate`
- **gRPC** — клиент к сервисам `auth_users`

## API

### Публичные эндпоинты (без авторизации)

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/room-types` | Список типов комнат |
| `GET` | `/cities` | Список городов с активными комнатами |
| `GET` | `/rooms` | Список комнат с фильтрами и пагинацией |
| `GET` | `/rooms/:id` | Информация о комнате |
| `GET` | `/rooms/:id/participants` | Участники комнаты |

### Защищённые эндпоинты (Bearer JWT)

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/rooms` | Создать комнату |
| `PUT` | `/rooms/:id` | Обновить комнату (только владелец) |
| `DELETE` | `/rooms/:id` | Удалить комнату (только владелец) |
| `POST` | `/rooms/:id/join` | Вступить в комнату |
| `POST` | `/rooms/:id/leave` | Покинуть комнату |
| `GET` | `/users/me/rooms` | Мои комнаты |

### Пагинация

`GET /rooms` поддерживает параметры:

| Параметр | Описание |
|----------|----------|
| `city` | Фильтр по городу |
| `type` | Фильтр по типу комнаты |
| `limit` | Размер страницы (по умолчанию `20`, максимум `100`) |
| `last_id` | Курсор — ID последней комнаты с предыдущей страницы |

Ответ содержит поле `next_cursor`. Передай его как `?last_id=` для получения следующей страницы.

## Система подписок

Уровень подписки определяет что может делать пользователь:

| Уровень | Комнат можно создать | Максимум участников |
|---------|---------------------|---------------------|
| `default` | 3 | 10 |
| `vip` | 5 | 20 |
| `super` | 10 | 50 |

Уровень подписки запрашивается из `auth_users` через gRPC при каждом создании и обновлении комнаты. При превышении лимитов возвращается `409 Conflict`.

## Типы комнат

Заполняются при миграции:

`бары` · `клубы` · `настолки` · `прогулка` · `посидеть` · `спорт` · `работа` · `квесты`

Актуальный список всегда доступен через `GET /room-types`.

## Конфигурация

Заполни `internal/config/config.env`. В продакшене переменные передаются через окружение.

| Переменная | Описание |
|------------|----------|
| `REST_ROOMS_HOST_ADDRESS` | Адрес сервера (например `:8083`) |
| `DB_HOST` / `DB_PORT` / `DB_USER` / `DB_PASSWORD` / `DB_NAME` | Подключение к PostgreSQL |
| `AUTH_GRPC_ADDR` | Адрес Auth gRPC сервиса (например `localhost:50051`) |
| `USER_GRPC_ADDR` | Адрес User gRPC сервиса (например `localhost:50052`) |
| `SERV_TIMEOUT` | Таймаут graceful shutdown (по умолчанию `5s`) |
| `LOG_LEVEL` | `local` / `dev` / `prod` |

## Миграции

```bash
go run ./cmd/migrator
```

Миграции находятся в `migrations/`, выполняются по порядку. Каждая миграция имеет файлы `up` и `down`.

## Запуск локально

```bash
# 1. Запустить PostgreSQL
docker run -d -p 5432:5432 -e POSTGRES_USER=user -e POSTGRES_PASSWORD=pass -e POSTGRES_DB=rooms postgres:16

# 2. Убедиться что auth_users запущен (нужны gRPC эндпоинты)

# 3. Заполнить конфиг
# отредактировать internal/config/config.env

# 4. Применить миграции
go run ./cmd/migrator

# 5. Запустить
go run ./cmd/app
```

## Сборка

```bash
go build -o rooms ./cmd/app
go build -o migrator ./cmd/migrator
```
